# From Research To Starter

## Purpose

This repository turns the SearXNG agent-search research work from `awesome-agent-search` into a small installable Claude Code starter.

The raw research report is not copied here. It may contain environment-specific details, local paths, or private operational context. Only public-safe, durable installation patterns belong in this repository.

## What Stays In `awesome-agent-search`

`awesome-agent-search` remains the source of truth for:

- adapter comparisons
- search-backend tradeoffs
- privacy principles
- hardening checklists
- evaluation rubrics
- dated research observations
- cross-agent patterns for Codex, Claude Code, OpenClaw, and generic agents

Use it to decide what is safe and worth standardizing.

## What Belongs Here

`oh-my-agent-search` should contain only the pieces needed to install and verify a working setup:

- Claude Code MCP setup commands
- SearXNG JSON endpoint verification
- dry-run-first installer behavior
- local readiness checks
- public-safe instruction templates
- smoke tests and negative tests
- troubleshooting for common setup failures

Use this repository to execute the chosen path.

## Installation Content Extraction Rules

When extracting installation material from a research report:

1. Replace real endpoints with `https://search.example.org`.
2. Remove local absolute paths, proxy values, account identifiers, and credentials.
3. Convert screenshots or one-off notes into commands that can be reviewed.
4. Separate observations from defaults.
5. Prefer dry-run commands before any command that mutates Claude Code configuration.
6. Add a smoke test for the happy path.
7. Add a negative test for private-context leakage.
8. Link back to the relevant research note in `awesome-agent-search`.

## Current Installable Path

The first installable path is intentionally narrow:

```text
Claude Code -> local stdio MCP adapter -> trusted or local SearXNG endpoint -> public search engines
```

The current default command shape is:

```sh
claude mcp add -s local -e SEARXNG_URL=https://search.example.org -t stdio searxng -- npx -y mcp-searxng
```

This command is generated by:

```sh
make install-preview URL=https://search.example.org
```

It can be checked against the endpoint before installation:

```sh
make install-preview-check URL=https://search.example.org
```

It is applied only with:

```sh
make install-apply-check URL=https://search.example.org
```

## Not Yet Included

Do not add these until they have equivalent verification:

- Codex install automation
- OpenClaw install automation
- remote HTTP MCP server deployment
- managed team setup
- automatic project-scope `.mcp.json` writes
- random public SearXNG instance fallback
- benchmark claims without dated sources

## Acceptance Criteria

A research-to-starter change is ready when:

- `make review` passes
- endpoint examples are placeholders
- installer behavior is dry-run by default
- smoke and negative tests are documented
- no private endpoints, local absolute paths, credentials, cookies, tokens, or session material are present
